### GET /api/health

**Purpose:** Check whether the ML model and grid data are loaded and the server is ready.

**Request:**
```
GET /api/health
```
No parameters, no body.

**Response `200 OK`:**
```json
{
  "status": "ok",
  "model_loaded": true,
  "grid_loaded": true
}
```

| Field | Type | Description |
|---|---|---|
| `status` | `string` | Always `"ok"` if the server is reachable |
| `model_loaded` | `boolean` | `true` when the ML model is loaded |
| `grid_loaded` | `boolean` | `true` when the grid reference data is loaded |

**Frontend Usage:**
- Poll this endpoint on page load.
- Disable the forecast button if `model_loaded` is `false`.
- Display a loading/warning banner if either field is `false`.

---

### GET /api/locations

**Purpose:** Autocomplete location search. Returns up to `N` matching places within the configured country.

**Request:**
```
GET /api/locations?q=<search_query>
```

| Parameter | Type | Required | Constraints | Description |
|---|---|---|---|---|
| `q` | `string` | ✅ Yes | min 3 chars | Location search string typed by the user |

**Response `200 OK`:**
```json
[
  {
    "place": "Mumbai, Maharashtra, India",
    "lat": 19.0760,
    "lon": 72.8777
  },
  {
    "place": "Mumbai Suburban, Maharashtra, India",
    "lat": 19.1300,
    "lon": 72.8600
  }
]
```

| Field | Type | Description |
|---|---|---|
| `place` | `string` | Full formatted address string — display in dropdown |
| `lat` | `float` | Latitude of the location |
| `lon` | `float` | Longitude of the location |

**Errors:**

| HTTP Status | Condition | `detail` |
|---|---|---|
| `400` | `q` < 3 characters | `"Query must be at least 3 characters."` |
| `500` | Geoapify API failure | Internal error message |

**Frontend Usage:**
- Trigger after the user types ≥ 3 characters (debounce: ~300 ms).
- Render each result in a dropdown as `place`.
- On selection, store `{ place, lat, lon }` in component state — all three are required for the forecast request.

### POST /api/forecast

**Purpose:** Run 7-day rainfall prediction for a selected location and start date.

**Request:**
```
POST /api/forecast
Content-Type: application/json
```

**Request Body (`ForecastRequest`):**
```json
{
  "location": "Mumbai, Maharashtra, India",
  "lat": 19.0760,
  "lon": 72.8777,
  "date": "2025-06-01"
}
```

| Field | Type | Required | Constraints | Description |
|---|---|---|---|---|
| `location` | `string` | ✅ Yes | 3–200 chars, non-blank | Full location name (from autocomplete `place`) |
| `lat` | `float` | ✅ Yes | −90 to 90 | Latitude (from autocomplete `lat`) |
| `lon` | `float` | ✅ Yes | −180 to 180 | Longitude (from autocomplete `lon`) |
| `date` | `string` | ✅ Yes | `YYYY-MM-DD` format | Forecast start date |

**Response `200 OK` (`ForecastResponse`):**
```json
{
  "location": "Mumbai, Maharashtra, India",
  "coordinates": {
    "lat": 19.0760,
    "lon": 72.8777
  },
  "forecast": [
    {
      "date": "2025-06-01",
      "rainfall_mm": 12.45,
      "status": "Moderate Rain"
    },
    {
      "date": "2025-06-02",
      "rainfall_mm": 0.03,
      "status": "No Rain"
    },
    {
      "date": "2025-06-03",
      "rainfall_mm": 1.80,
      "status": "Light Rain"
    },
    {
      "date": "2025-06-04",
      "rainfall_mm": 23.10,
      "status": "Heavy Rain"
    },
    {
      "date": "2025-06-05",
      "rainfall_mm": 8.60,
      "status": "Moderate Rain"
    },
    {
      "date": "2025-06-06",
      "rainfall_mm": 0.00,
      "status": "No Rain"
    },
    {
      "date": "2025-06-07",
      "rainfall_mm": 2.10,
      "status": "Light Rain"
    }
  ]
}
```

**Response Fields:**

| Field | Type | Description |
|---|---|---|
| `location` | `string` | Echo of the requested location name |
| `coordinates.lat` | `float` | Latitude |
| `coordinates.lon` | `float` | Longitude |
| `forecast` | `ForecastDay[]` | Array of exactly **7** daily predictions |
| `forecast[].date` | `string` | `YYYY-MM-DD` |
| `forecast[].rainfall_mm` | `float` | Predicted rainfall in millimetres (≥ 0) |
| `forecast[].status` | `string` | Human-readable category (see §8) |

**Errors:**

| HTTP Status | Condition | `detail` |
|---|---|---|
| `400` | Validation failure (missing field, bad date, short location, out-of-range lat/lon) | Pydantic error description |
| `400` | Business logic error (e.g. model not found) | Descriptive string |
| `422` | Pydantic schema validation failure | FastAPI default validation error body |
| `500` | Unexpected server error | `"Internal server error during forecast generation."` |

## 6. Data Contracts (Schemas)

### ForecastRequest
```python
class ForecastRequest(BaseModel):
    location: str          # min 3, max 200 chars; stripped of whitespace
    lat:      float        # -90 ≤ lat ≤ 90
    lon:      float        # -180 ≤ lon ≤ 180
    date:     date         # YYYY-MM-DD
```

### LocationSuggestion
```python
class LocationSuggestion(BaseModel):
    place: str
    lat:   float
    lon:   float
```

### ForecastResponse
```python
class Coordinates(BaseModel):
    lat: float
    lon: float

class ForecastDay(BaseModel):
    date:        str    # YYYY-MM-DD
    rainfall_mm: float  # ≥ 0
    status:      str    # See §8

class ForecastResponse(BaseModel):
    location:    str
    coordinates: Coordinates
    forecast:    List[ForecastDay]  # len == 7
```

## 7. Internal Service Pipeline

When `POST /api/forecast` is called, the backend executes:

```
ForecastRequest
      │
      ▼
GridService.get_nearest_grid(lat, lon)
      │  Returns: { grid_id, center_lat, center_lon }
      ▼
DateService.generate_7_day_forecast(start_date)
      │  Returns: [date_obj × 7]
      ▼
for each date:
  ModelService.predict_single_day(grid_id, center_lat, center_lon, date)
        │
        ├─ Look up historical features for grid_id in master_df
        │   Features: hem, wind_speed, uth, olr, lst_k, cer, cot
        ├─ Add cyclic temporal encodings:
        │   day_sin, day_cos, week_sin, week_cos
        ├─ Scale with fitted StandardScaler
        ├─ ML model.predict()  → log-scale rainfall
        ├─ np.expm1()          → real-scale rainfall (mm)
        └─ PhysicsConstraints.apply_post_inference_adjustments()
              Inputs: latitude, longitude, month, olr, uth, elevation
              Returns: (final_rain_mm, metadata)
      │
      ▼
ForecastResponse { location, coordinates, forecast: [ForecastDay × 7] }
```

## 8. Rainfall Categories

The `status` field in each `ForecastDay` is derived from `rainfall_mm`:

| `status` | `rainfall_mm` range | Suggested UI Colour / Icon |
|---|---|---|
| `"No Rain"` | `< 0.1 mm` | ☀️ Blue-sky / dry |
| `"Light Rain"` | `0.1 – 2.49 mm` | 🌦️ Light blue |
| `"Moderate Rain"` | `2.5 – 14.99 mm` | 🌧️ Mid blue |
| `"Heavy Rain"` | `≥ 15 mm` | ⛈️ Dark blue / red |

## 9. Error Handling Guide

All API errors follow a consistent shape:

```json
{
  "detail": "<human-readable error message>"
}
```

**Recommended frontend behaviour:**

| Status | User-facing action |
|---|---|
| `400` | Show inline validation error near the offending field |
| `422` | Show a generic "Please check your inputs" message |
| `500` | Show a toast / banner: "Something went wrong, please try again" |
| Network / CORS | Show "Could not reach the server" |

## 10. Frontend Integration Patterns

### A. Page Load

```js
// 1. Health check — gate the UI
const health = await fetch('/api/health').then(r => r.json());
if (!health.model_loaded) {
  showBanner('Model is loading, please wait…');
  disableForecastButton();
}
```

### B. Location Autocomplete

```js
// Debounce input, fire when query.length >= 3
const suggestions = await fetch(`/api/locations?q=${encodeURIComponent(query)}`)
  .then(r => r.json());

// Each item: { place, lat, lon }
renderDropdown(suggestions);
```

### C. Forecast Request

```js
const body = {
  location: selectedPlace,   // string from autocomplete
  lat: selectedLat,          // float
  lon: selectedLon,          // float
  date: "2025-06-01"         // YYYY-MM-DD string
};

const response = await fetch('/api/forecast', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(body)
});

const data = await response.json();
// data.forecast → array of 7 ForecastDay objects
renderForecastCards(data.forecast);
```

### D. Forecast Card Rendering

Each `ForecastDay` should render:
- **Date** — format with `new Date(day.date)` for locale-aware display
- **Rainfall** — `day.rainfall_mm` mm (round to 1 decimal)
- **Status badge** — styled by `day.status` category (see §8)
- Optional: a bar chart scaled to `max(rainfall_mm)` across 7 days

## 11. Environment Variables

Defined in `.env` at project root and loaded via `backend/core/config.py`:

| Variable | Description | Default |
|---|---|---|
| `GEOAPIFY_API_KEY` | Geoapify autocomplete API key | *(required)* |
| `COUNTRY_CODE` | ISO 3166-1 alpha-2 to filter results (e.g. `in`) | `in` |
| `DEFAULT_LIMIT` | Max location suggestions per query | `5` |
| `MIN_QUERY_LENGTH` | Minimum chars before `/api/locations` fires | `3` |
